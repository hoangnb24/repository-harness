# VSF Data Profiler v0.1 MVP

## Summary

This release introduces a local CLI for profiling CSV datasets against a DBML
schema contract, detecting data quality and relationship issues, running
target-based association analysis, and generating Markdown/HTML reports.

The final demo package is documented at
`docs/demo/vsf-data-profiler.md`. It includes a 5-10 minute demo script,
command checklist, artifact tour, fake LLM path, OpenAI smoke path, and
guardrail fallback caveats.

## Features

- Parse DBML tables, columns, primary keys, unique constraints, not-null
  constraints, foreign keys, short `Ref:` syntax, `Ref` blocks, quoted and
  schema-qualified identifiers, and composite primary/unique indexes.
- Generate `schema_parse_report.json` with parser diagnostics and unsupported
  construct warnings.
- Catalog CSV files and map file stems to DBML tables.
- Add optional Postgres connector mode for selected local database tables,
  chunked extraction into DuckDB-readable scans, and additive
  `connector_metadata.json`.
- Profile CSV tables with DuckDB.
- Generate table and column summaries.
- Detect schema and data quality issues.
- Validate primary keys and foreign keys.
- Generate issue catalog entries with severity, evidence SQL, sample paths,
  probable causes, and suggested fixes.
- Generate `schema_diagram.dbml` and `schema_diagram.json` with a dbdiagram.io
  embed link, PK/FK details, relationships, and CSV-to-table mapping.
- Add a static web UI prototype for DBML upload, multi-CSV upload, CSV-to-table
  linking, PK/FK review, and dbdiagram.io visualization preview.
- Add a local-only web runner with upload mode for demo/small-medium
  DBML/CSV/rules jobs and local path mode for larger local datasets. Both modes
  call the existing Python DuckDB pipeline and display runtime artifacts.
- Add an interactive local web-runner dashboard that renders existing chart
  specs and machine artifacts from protected artifact URLs. Static
  `report.html` keeps its deterministic Visual Summary.
- Run target-based influence analysis labelled as association, not causation.
- Generate Markdown and HTML reports.
- Optionally generate guarded L4 Senior Data Scientist narratives from
  structured artifacts only, with fake local validation and opt-in OpenAI
  provider support.
- Provide synthetic demo data with injected data quality issues.
- Provide Olist schema/rules and demo commands for Kaggle-authenticated users.

## Known Limitations

- No hosted production web UI or hosted backend job runner; the browser
  workspace and local runner are local-first only.
- Browser upload mode is for demo/small-medium files; use local path mode or
  the CLI for larger local datasets.
- Postgres is the only live database connector in this MVP.
- No production database monitoring.
- No Kafka, Spark, or realtime processing.
- No full lineage graph.
- Olist download requires Kaggle CLI authentication.
- Influence analysis reports association, not causation.
- OpenAI L4 output may fall back deterministically when guardrails reject
  unsupported numbers or references.
