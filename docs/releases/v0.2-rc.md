# VSF Data Profiler v0.2 Release Candidate

Date: 2026-06-16

## Purpose

This release candidate packages the current local VSF Data Profiler feature set
behind one reproducible validation path. The product framing remains generic
CSV plus DBML/schema Smart EDA for data scientists; package export, connectors,
dashboard review, benchmarks, and L4 narrative are optional advanced surfaces.

US-050 remains the original release-candidate hardening base. US-052 added the
MySQL/MariaDB connector and optional PDF export gate. US-053 is the
release-freeze cleanup gate for documentation, version metadata, deployment
boundary wording, generated-file hygiene, and Harness evidence. US-055 is the
product capability gate for deterministic per-table assessment and analysis
impact reporting. US-056 is the local web-runner experience gate for the
optional Smart EDA console and post-run Table Assessment dashboard.

## v0.2.0-rc2 Highlights

`vsf-profiler-v0.2.0-rc2` publishes the US-055 and US-056 increments:

- `table_assessments.json` is now a canonical deterministic artifact with one
  readiness, health, role, relationship-risk, and analysis-impact summary per
  profiled table.
- Markdown/HTML reports, export packages, artifact audit, and the web-runner
  dashboard link the table assessment artifact without renaming existing
  outputs.
- Optional L4 narrative context includes table assessments as bounded evidence,
  and guardrails reject unsupported table or analysis-impact claims.
- The local web runner is redesigned as a restrained Smart EDA console with
  a run-first flow, clear `127.0.0.1` local backend boundary, and dedicated
  Table Assessment dashboard section.
- Desktop and mobile dashboard screenshots are captured by the Playwright
  validation path as release evidence when local browser tooling is available.

## Deployment Boundary

The Vercel deployment is a static preflight surface only. It serves the
browser-side DBML/CSV mapping and visualization UI from `web/`; it does not
host the Python/DuckDB profiler, database connectors, local path jobs, LLM
narrative generation, package/PDF export, or the web-runner dashboard backend.

Full browser-driven profiling jobs require the local runner:

```bash
make web-runner
open http://127.0.0.1:8765
```

The local runner binds to `127.0.0.1` and writes web job artifacts under
`outputs/web_runs/<job_id>/artifacts`.

The local web runner is the demo console for browser-driven profiling. The
first screen keeps upload mode and local path mode separate, starts the existing
Python/DuckDB pipeline, streams runtime events, and then makes the post-run
dashboard the optional review surface for readiness, issue counts, Table Assessment,
graphs, and artifact links.

## Environment Doctor

```bash
vsf-profiler doctor
```

Expected behavior:

- required Python/package/DuckDB checks report `ok`;
- optional Postgres, MySQL/MariaDB, PDF backend, OpenAI, Node, and Playwright
  checks report `ok`, `missing`, or `skipped`;
- environment secret values are never printed. Presence is reported with
  redacted wording such as `OPENAI_API_KEY present (redacted)`.

## One-Command Demo

```bash
make demo-full
```

This command runs:

1. `vsf-profiler doctor`
2. `make demo-small`
3. `vsf-profiler package --input outputs/demo_small --output outputs/demo_small_package --zip --pdf --force`
4. `python scripts/verify_vsf_artifacts.py --run-dir outputs/demo_small --package-dir outputs/demo_small_package --zip-path outputs/demo_small_package.zip`
5. `npm run test:e2e:dashboard` when local Node and Playwright are installed

If local Playwright is absent, the target prints an explicit skip message and
continues after the required profiler/package/audit checks pass.

## Final Audit

Run the audit directly when validating an existing output:

```bash
python scripts/verify_vsf_artifacts.py \
  --run-dir outputs/demo_small \
  --package-dir outputs/demo_small_package \
  --zip-path outputs/demo_small_package.zip
```

The audit fails on:

- missing canonical JSON/report/chart/runtime artifacts;
- raw CSV files outside `samples/`;
- connector temporary extracts;
- unredacted connection URLs, passwords, tokens, bearer tokens, or OpenAI-style
  keys in generated text artifacts;
- noncanonical `run_summary.json` artifact path keys;
- package manifest redaction status other than `passed`.

## Large Dataset Benchmark

CI-safe benchmark:

```bash
make benchmark-small
python -m json.tool outputs/benchmark_ci/run/performance_guard_report.json
```

Optional larger local benchmark:

```bash
make benchmark-large
python -m json.tool outputs/benchmark_large/run/performance_guard_report.json
```

`performance_guard_report.json` records environment-specific evidence:

- total and per-table generated rows;
- runtime stage timings from `run_summary.json`;
- run event count from `run_events.jsonl`;
- peak RSS memory when supported;
- output artifact sizes;
- bounded influence row and feature limits;
- default Postgres chunk size and CSV DuckDB scan mode;
- chart, report, package, and artifact audit success;
- source scan results for production `pandas.read_csv` and unguarded
  `.fetchdf()` usage.
